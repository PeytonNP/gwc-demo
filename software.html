Unit Testing
Testing a single component, function, or module by itself. Here are some java examples.

Test cases can be designed in many ways. It is common practice to say a test case either "passes" or "fails". If the test case passes, this means your program received the expected result.

assertEquals(0, 0); // Does 0 = 0? YES! This test case will PASS!
assertEquals(0, 5); // Does 0 = 5? NO! This test case will FAIL!
assertTrue(0); // Does 0 = true? NO! This test case will FAIL! 0 means FALSE
assertTrue(1); // Does 1 = true? YES! This test case will PASS! 1 means TRUE
More on Testing
Integration Testing
Testing if different modules work as expected when you combine them together.

Integration testing is important when your application uses many different components.


Let's say a python program adds this as a new row to a database: 1 | Welcome! | Success
Now I want to get all the rows in the database. This is what is returned to my program:
100 | Welcome Girls Who Code! | FAILURE
As you can see the two rows are not the same! This would FAIL an integration test!

Open Source, Libraries, and Frameworks
A lot of functionality already exists! Why reinvent the wheel? Everything doesn't need to be created from scratch.

Open Source projects allow anyone to contribute or use existing code in other projects. Firebase SDK is an open-source project initially created and backed by Google. Anyone can add to the project!

Libraries are a collection of resources that are specifically made to be used in other applications. The Apache Commons library has a ton of functionality to do things like sending emails or parsing strings.

A framework is a pre-written structure that developers can use as a foundation to build on top of. Angular JS and React Native are examples popular front-end frameworks.

Cloud Computing
The cloud is a network of computers that can be accessed over the internet!

iCloud, Google Drive, and Dropbox are common consumer-driven cloud storage solutions. When we think of the Cloud from a development point of view, we tend to think of services. As a developer, I want to make my life as easy as possible. One way to do that is to use cloud solutions! Let's say my team needs a database but doesn't want to spend the time creating one and maintaining it ourselves. One easy way to get around this is to use a cloud provider. Now let's say that team needs to perform complex calculations but their laptops are not very powerful. We can rent a computer or resources from a cloud provider to do the work for us.

Here are some popular examples of cloud providers: IBM Cloud, Amazon Web Services (AWS), Google Cloud Platform, and Microsoft Azure

Front-End and UI/UX Design

The role of a front-end developer is vital to an application's success. Front-end developers research and design the ins and outs of how a user will use and interact with software. For example, someone who uses a smartphone or tablet will likely interact with applications via touch. A user who accesses a desktop application will likely use a mouse, trackpad, keyboard, or other physical device to navigate the application. Someone who accesses a web application may use the application in a combination of ways. This can become even more complex if you introduce the use of a stylus. It is the job of front-end developers to make sure end-users (people who use the software or application) can quickly and easily use the service. Would you use any software, apps, or websites that are not easy to use?

Back-End

There are various ways back-end development can be described. One relatively easy way to think about back-end development is to focus on functionality and logic. Back-end developers tend to design code that 1. Works as Expected 2. Is Logical 3. Performs as Needed. Let's try to break that down a bit more. Let's say you downloaded an app that gives step-by-step directions to any location the user searches. If the application doesn't provide directions, then it does not work as expected; leading to a poor experience for the user. What does it mean for an application to be logical? Let's say I am user based in the United States and want to search for clothing stores in the state of Illinois. It is not logical for the application to return a list of clothing stores in Canada. Similarly, it is not logical to return directions to the user that require them to do more work. Let's say I get directions to point A that requires 10 steps. But the easiest route is 5 steps. Which route do you think the user would want? Scenarios such as these can also lead to a poor user experience. Lastly, let's discuss performance. If I want to learn how to get from point A to point B, I could use a paper map. Why would I do that? It's a slow, inconvenient process which requires me to do a lot manual work. That's where software comes in. However, if the application takes 5 minutes (or even more than 1-2 minutes) to find the best directions to place near me, the application does not perform well.


Data Storage Solutions

Almost all applications use data in some way or another. A data storage solution is a way to manage all the information needed to make a an application work as expected. Recall you added yourself as a user on the homepage and may have posted a few comments. All of that information you the user provided needs to be stored somewhere where it can be easily accessed. The simplest way to think about data is to visualize rows in a table. The first image above shows what the database schema may look like. Click here to learn more about database schemas. In short, a schema simply designs the database table by describing the columns in a table. You can see the USER table has a column named FIRST_NAME. Now take a look at the second image. It looks just like a normal table! This is how many databases are structured. These are called relational databases. The application you are using now uses a different type of database called a NoSQL Database. These databases are bit different. Instead of columns in a table, NoSQL databases uses (key, value) pairs to store data. Click here to learn more about SQL vs NoSQL databases. You will notice the image below looks quite different, almost like a tree. You can see that (key, value) pairs are used instead of columns. In the blue box, you can see we have a key called "message" and a value called "Hello World! Welcome Girls Who Code Participants!" We can follow this pattern with any and all information we want to store. Voila! Now we have a database.
